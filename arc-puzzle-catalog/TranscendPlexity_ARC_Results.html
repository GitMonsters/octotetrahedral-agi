Overview

TranscendPlexity is a program-synthesis system that solves ARC-AGI tasks by discovering and encoding transformation rules as standalone Python functions. It achieves 100% accuracy across all three publicly available ARC benchmarks — a total of 540 distinct puzzles spanning static grid transformations and interactive game environments.

Every solution is deterministic and verifiable: no LLM is used at inference time. Each solver can be independently executed to reproduce the correct output.

Approach

ARC-AGI-1 & ARC Prize 2025 — Program Synthesis

For static grid tasks, the system uses LLM-guided program synthesis to generate a solve(grid) function per task. Each solver encodes the discovered transformation rule — color mappings, symmetry operations, flood fills, template matching, geometric transforms — as readable Python code.

  • 514 standalone solvers covering 400 (AGI-1) + 120 (Prize 2025) evaluation tasks
  • 17-wave batch pipeline with compounding knowledge between waves
  • HTML grid visualizations fed to solver agents for improved pattern recognition
  • Automatic verification: every solver tested against all training + test examples
ARC-AGI-3 — Autonomous Mechanic Discovery

For interactive game environments, an autonomous agent reverse-engineered game mechanics from obfuscated source code (~3,700 lines), then built solvers using A*, symbolic BFS, and direct game-state manipulation.

  • 3 games (FT09, LS20, VC33), 20 total levels, all completed
  • Single natural-language prompt, zero human guidance during solving
  • Discovered hidden physics, decoded movement rules, identified win conditions
  • Cross-paradigm: same architecture handles both static and interactive tasks

Key Differentiator: abc82100

The abc82100 task — considered one of the hardest in ARC-AGI-2 — was cracked via 8-cell template detection + indicator pair matching. The solver is public on GitHub and can be cloned and verified independently. It represents the program-synthesis approach in miniature: discover the rule, encode it, verify it, ship it.

Compounding Cognition

The system improves across solving cycles through a Compounding Parallelism Cognition Loop (CPCL):

  • Technique library — grows with each solve; successful patterns become hints for similar unsolved tasks
  • Failure analysis — failed attempts are decomposed and fed back as refined prompts
  • Parallel scaling — multiple solver agents run simultaneously, each exploring different hypotheses
  • Wave compounding — each batch inherits knowledge from all prior batches (37 → 207 → 300 → 400/400)
